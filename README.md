**About**

As a Windows developer managing repetitive interface operations in corporate environments, I built Macro Recorder Studio to reduce time spent on routine tasks. I grew tired of manually repeating form entries and validation steps during daily workflows and software testing cycles. The goal was to create a dependable automation utility that supports consistent execution of office procedures and testing routines without requiring custom scripting.

**What it does**
- Record mouse + keyboard sequences for repetitive office tasks
- Replay forms / data entry workflows
- Automate UI testing scenarios
- Speed up software demos and tutorials
- Schedule recurring computer tasks (backups, file moves)
- Capture step-by-step procedures for training documentation

**Features**
- Variable playback speed (0.25x–4x)
- Randomize timing (for natural replay)
- Schedule recordings
- Hotkey controls
- Encrypted profile storage
- Export/import .macro files

**Requirements**
Windows 10/11 x64, .NET Desktop Runtime 6.0+, 2 GB RAM

**FAQ**
- Can I share macro files with my team? (Yes, export/import .macro files)
- Does it work in remote desktop sessions? (Yes, runs locally and replays through the session)
- Can I encrypt sensitive recordings (e.g., contains passwords)? (Yes, AES-256 profile encryption)
- How do I trigger a macro from a specific application? (Use the per-app profile feature in Settings)
